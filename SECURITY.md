# Security Policy

Thank you for helping keep **Urban Stack** secure.

The security and reliability of Urban Stack are essential for municipalities, public institutions, and all organizations that rely on the platform. We appreciate responsible disclosure of potential security vulnerabilities.

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not report it publicly** through GitHub issues, discussions, or other public channels.

Instead, please contact the Urban Stack maintainers directly:

📧 **[urbanstack@guetersloh.de](mailto:urbanstack@guetersloh.de)**

Please include:

* A description of the vulnerability
* Steps to reproduce the issue
* Potential impact
* Suggested remediation (if available)

## Responsible Disclosure

We are committed to working with security researchers and contributors to investigate and resolve security issues in a timely and responsible manner.

When reporting a vulnerability, we ask that you:

* Give us reasonable time to investigate and resolve the issue
* Avoid publicly disclosing the vulnerability before a fix is available
* Provide sufficient information to help us reproduce and validate the issue

## What to Expect

After receiving a report, we will:

1. Acknowledge receipt of the report
2. Assess the severity and impact
3. Investigate and validate the issue
4. Develop and test a fix
5. Coordinate disclosure where appropriate

## Scope

This policy applies to:

* Urban Stack Core Platform
* Urban Data Hub
* Urban Gov Hub
* Urban Citizen Hub
* Official Urban Stack modules and extensions

## Our Commitment

Urban Stack is committed to:

* Secure software development
* Open and transparent collaboration
* Protection of municipal systems and data
* Responsible vulnerability management

We appreciate the efforts of security researchers, contributors, and community members who help improve the security of the Urban Stack ecosystem.
